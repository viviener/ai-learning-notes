# 第4课：数据库（Database）到底是什么？

这一课非常重要。

因为以后无论你看到：

```text
Agent
MCP
API
ChatGPT
Claude
企业系统
网站
App
```

背后几乎都会有数据库。

---

# 先说结论

一句话：

> 数据库 = 专门存放和管理数据的系统。

---

# 先从现实世界开始

假设你开了一家书店。

每天都会产生：

```text
客户信息
订单信息
图书信息
库存信息
```

---

如果全部记在纸上：

```text
纸1
纸2
纸3
纸4
```

很快就乱了。

---

于是你买了一个档案柜。

里面有：

```text
客户抽屉
订单抽屉
图书抽屉
库存抽屉
```

---

这就是数据库最简单的理解：

> 有组织地存放数据的地方。

---

# 为什么普通文件不够？

这是数据库出现的原因。

---

假设你有：

```text
客户名单.xlsx
```

里面：

```text
张三
李四
王五
```

---

刚开始没问题。

---

一年后：

```text
10万个客户
```

---

你想找：

```text
最近30天买过书的人
```

---

Excel 开始变慢。

管理开始混乱。

---

于是出现了：

```text
Database（数据库）
```

---

# 数据库解决什么问题？

它主要解决：

## 存

例如：

```text
用户资料
订单
聊天记录
```

---

## 查

例如：

```text
找到张三

找到昨天的订单

找到最近聊天
```

---

## 改

例如：

```text
修改手机号

修改地址
```

---

## 删

例如：

```text
删除无效数据
```

---

软件行业通常叫：

```text
CRUD
```

即：

```text
Create
Read
Update
Delete
```

---

你不用记英文。

记住：

```text
增
查
改
删
```

即可。

---

# ChatGPT 背后有数据库吗？

当然有。

否则会发生什么？

---

你今天问：

```text
你好
```

---

一分钟后：

```text
你是谁？
```

---

系统会回答：

```text
我不知道
```

---

因为它没保存数据。

---

所以背后一定有数据库存储：

```text
账户
聊天记录
项目
文件
```

等等。

---

# Agent 为什么需要数据库？

假设未来你做一个：

```text
投资分析Agent
```

---

第一天：

```text
分析苹果公司
```

---

第二天：

```text
分析微软
```

---

第三天：

```text
请比较苹果和微软
```

---

如果 Agent 不记得前两天的结果：

```text
无法比较
```

---

所以：

```text
Agent
↓
数据库
```

几乎是天然搭档。

---

# Repo 和数据库有什么区别？

这是很多人容易混淆的。

---

## Repo

存放：

```text
代码
文档
配置
项目文件
```

---

例如：

```text
README.md

Docker.md

main.py
```

---

这些是：

```text
项目资产
```

---

## 数据库

存放：

```text
用户
订单
聊天
记录
```

---

这些是：

```text
运行数据
```

---

所以：

```text
Repo
≠
数据库
```

---

关系更像：

```text
Repo
=
餐厅菜单

数据库
=
顾客订单
```

---

菜单不会天天变。

---

订单每天都在变。

---

# 对你目前最重要的理解

不要把数据库理解成：

```text
一个软件
```

---

更重要的是理解：

> 数据库是为了解决“大量数据如何长期保存和快速查询”这个问题。
