# 第5课：API 到底是什么？

这一课非常重要。

因为从这里开始，你会第一次看懂：

```text
ChatGPT 调用工具

Agent 调用服务

MCP 连接系统

第三方插件

自动化工作流
```

背后的共同机制。

---

# 先说结论

一句话：

> API = 软件之间沟通的标准接口。

---

# 从现实世界开始

假设你去餐厅。

你不会直接冲进厨房：

```text
找厨师
开冰箱
自己炒菜
```

---

而是：

```text
看菜单
↓
告诉服务员
↓
服务员通知厨房
↓
厨房做好
↓
服务员端回来
```

---

这里：

### 厨房

```text
真正干活的系统
```

---

### 服务员

```text
API
```

---

### 你

```text
调用方
```

---

所以：

> API 就像服务员。

---

# 为什么需要 API？

因为系统之间不能直接乱访问。

---

假设：

```text
ChatGPT
```

想知道：

```text
今天苹果股价
```

---

它不能：

```text
直接进入证券公司数据库
```

---

而是：

```text
请求API
↓
获取结果
```

---

# API 的本质

其实就是：

```text
我按照什么格式提问

你按照什么格式回答
```

---

双方都遵守规则。

---

这套规则：

```text
API
```

---

# 为什么 AI 世界离不开 API？

因为 AI 本身不知道现实世界。

---

例如：

你问：

```text
今天波士顿天气如何？
```

---

ChatGPT 不会自己去测天气。

---

而是：

```text
天气API
↓
返回天气
↓
AI解释给你
```

---

所以：

```text
AI
+
API
```

才真正有用。

---

# 一个关键理解

很多人以为：

```text
ChatGPT
```

无所不能。

---

实际上：

```text
ChatGPT
=
大脑
```

---

而：

```text
API
=
感官和工具
```

---

# API 和数据库有什么区别？

数据库：

```text
存数据
```

---

API：

```text
取数据
```

---

关系：

```text
数据库
↓
API
↓
AI
```

---

所以：

```text
数据库
≠
API
```

---

# API 和 Agent 的关系

未来你会做：

```text
投资Agent
```

---

Agent 需要：

```text
查股价

查新闻

查财报
```

---

Agent 自己不会。

---

它会：

```text
调用API
```

---

所以：

```text
Agent
↓
API
↓
外部世界
```

---

# API 和 MCP 的关系

传统方式：

```text
Agent
↓
天气API

Agent
↓
股票API

Agent
↓
邮件API
```

---

MCP 出现以后：

```text
Agent
↓
MCP
↓
各种工具
```

---

所以：

> MCP 不是替代 API。

而是：

> 统一管理 API 和工具。

---
